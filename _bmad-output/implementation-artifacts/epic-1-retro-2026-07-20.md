### Recurring review feedback

- **Security paths need allowlist + adversarial matrix**, not regex blocklists (1.6 → element allowlist; 1.7 → SMIL/style bypass tests).
- **Test assertions must not distort product behavior** — layout probes should use inequalities or metadata, not force UX reorder (1.5).
- **Re-run turbo with `--force`** before story sign-off when cache timing suspicious (Epic 0 lesson, applied consistently).
- **Fixture bytes must decode** before embedding in story specs — validate with Typst or checksum-after generation (1.6).

### Breakthrough moments

- **Story 1.3:** Manifest-driven golden harness became the spine for all subsequent fixtures and CI.
- **Story 1.4:** In-job Docker build in `pdf-golden.yml` — PR Dockerfile is source of truth, not stale GHCR image.
- **Story 1.6:** Two sanitizer profiles established — logo allowlist vs Typst output strip-list; critical for 1.7 preview without corrupting `<use>` glyphs.
- **Story 1.8:** 100-iteration soak with zero drift proved determinism beyond single-run goldens.

## Key takeaways

1. **Adversarial review is load-bearing for security** — dev agents default to blocklists and test-shaped shortcuts; review caught bypass vectors that would have passed green CI.
2. **Tests must prove behavior, not reshape it** — strict equality assertions on layout metadata caused UX distortion in 1.5; relax probes or use bounded inequalities.
3. **Parallel test isolation beats directory-wide globs** — shared `.tmp/preview-*` assertions race under `bun test packages/render`; assert on the specific temp path under test.
4. **Spike PASS with honest CI-pending** — local-container evidence unblocks Epic 2; first-push re-verify list now includes pdf-golden and soak.
